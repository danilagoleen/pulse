Основные Музыкальные Лады и Шкалы
Музыкальные лады (modes) и шкалы (scales) — это последовательности нот, определяемые интервалами между ними. Вы упомянули диатонические моды (как Ionian, Dorian), моды вроде 4th (Lydian) и 5th (Mixolydian), а также пентатоники (M penta — Major Pentatonic, m penta — Minor Pentatonic) и блюзовые шкалы (mBlues — Minor Blues). "Полный список" всех возможных шкал бесконечен, поскольку музыканты создают новые вариации, но ниже я приведу полный список наиболее распространённых и стандартных шкал в западной музыке, включая диатонические моды, пентатоники, блюзовые, гармонические, мелодические и другие экзотические. Ваш вопрос касается аналогов круга Иттена (цветовые гармонии) и круга Камелот (гармоничные ключи для миксов) для музыкальных шкал (scales/modes). В музыке такие инструменты существуют в виде визуальных "колёс" (wheels) и математических моделей, которые помогают определять совместимость шкал по интервалам, общим нотам и консонансам. Это особенно полезно для полифонии (многоголосия), где сочетание шкал создаёт богатое, слоистое звучание без диссонансов (clashing notes).
Я опираюсь на стандартные концепции из теории музыки:

Circle of Fifths (Квинтовый круг) — база для многих "колёс", показывает отношения через квинты (интервал 3/2 в частотах).
Camelot Wheel — упрощённая версия для DJ, где ключи (основанные на major/minor scales) нумеруются (1-12, A/B для minor/major). Совместимость: смежные ключи (±1) или relative (A↔B в одном номере).
Для шкал: Фокус на modes (моды диатонической гаммы) и экзотических шкалах. Совместимость измеряется через shared intervals, rotations и consonance ratios.

Основные "Круги" для Сочетания Шкал
Эти инструменты визуализируют отношения, аналогично Иттену (комплементарные/аналоговые) или Камелот (гармоничные shifts).

Circle of Fifths (Квинтовый Круг) — Базовый для всех.
Структура: Круг с 12 тонами, где каждый шаг — квинта (7 полутонов). Внешний круг: major keys; внутренний: minor.
Сочетание шкал: Переход на ±1 (adjacent) для smooth modulation; +7 полутонов для relative major/minor. Для полифонии: Шкалы из смежных ключей делят 6/7 нот, создавая богатство без диссонанса (например, C Major + G Major для added fifths).
Пример для полифонии: В фуге Баха — modulation через fifths для layered voices.

Camelot Wheel (расширенно для modes).
Структура: Нумерованный круг (1-12A/B), основан на Circle of Fifths.
Сочетание шкал: Не только keys, но и modes (например, Mixolydian как "5th mode" major). Совместимость: ±1 (energy boost), same number A↔B (relative). Для экзотических: Добавьте blues/pentatonic как subsets.
Пример: 8A (A Minor) + 8B (C Major) для polyphonic layering — общие ноты, но different flavors (minor pentatonic over major для bluesy richness).

Harmonic Wheel (от harmonicwheel.com).
Структура: Круг, показывающий consonance (2-note intervals, 3-note chords). Включает major/minor scales, modes, key signatures, cycle of fifths.
Сочетание шкал: Показывает affine chords (общие с шкалой), pentatonic/diminished/hexatonic scales. Для полифонии: Выберите scales с max consonant intervals (например, major + harmonic minor для raised 7th в voice leading).
Пример: Major scale + relative minor для polyphonic canon; augmented/diminished chords как bridges для tension/resolution.

ScaleWheel (от musedlab.org).
Структура: Chromatic circle с интервалами; цвета для harmonic functions (root, third, etc.).
Сочетание шкал: Визуализирует interval structure; combine scales с similar colors/functions для harmony. Для modes: Rotate круг для modal shifts.
Пример для полифонии: Dorian + Mixolydian (shared intervals: W-H-W) для jazz polyphony — layered solos без clashing.

Mode Wheel (например, RubberDave's или аналогичные cardboard tools).
Структура: Два диска: внешний — modes (Ionian-Dorian-...-Locrian); внутренний — 12 нот. Rotate для relative modes.
Сочетание шкал: Находит equivalents (e.g., G Mixolydian = C Major notes). Для полифонии: Switch modes в разных voices (e.g., bass в Aeolian, melody в Dorian) для richness.
Пример: В прог-роке — polyphonic layers из diatonic modes для complex textures.

Musicians’ Theory Wheel (коммерческий инструмент).
Структура: Rotate для matching chords/scales; включает circle of fifths, progressions.
Сочетание шкал: Instant harmonic compatibility (e.g., scales с shared chords). Для полифонии: Combine via progressions (ii-V-I в different modes).


Другие: Chord Wheel (для chords over scales), Pentatonic Wheel (для blues/ethnic).
Математические Формулы для Сочетания Шкал
Музыка — это ratios частот и set theory (наборы нот как binary). Совместимость: Max overlap нот/intervals для consonance в полифонии.

Frequency Ratios (Pythagorean Tuning):
Базовые интервалы: Octave = 2/1, Fifth = 3/2, Fourth = 4/3, Major Third = 5/4.
Формула для scale generation: Start с root f, add multiples: f × (3/2)^n / 2^k (где k держит в octave).
Для сочетания: Scales совместимы, если ratios align (e.g., combine major + pentatonic: shared 1, 2, 3, 5, 6 steps).

Interval Vector/Spectrum (Forte/Hanson):
Scale как 12-bit binary (e.g., Major = 101010110101 в semitones).
Vector: Count intervals (p=perfect, m=minor, etc.): [intervals of 1,2,...,6 semitones].
Формула: Для scales A и B, compatibility = sum(min(A_vec[i], B_vec[i])) / max(len(A), len(B)).
Для полифонии: Shared vector → no clashing (e.g., all diatonic modes имеют vector [254361], combine freely для rich layers).

Rotations и Modes:
Mode = cyclic shift: Shift binary на k semitones.
Формула: New_scale = (old_scale << k) | (old_scale >> (12-k)) mod 2^12.
Для сочетания: Rotate до max overlap (Jaccard: |A ∩ B| / |A ∪ B| > 0.7 для good polyphony).

Reflections/Inversions:
Invert: Reverse intervals (e.g., Major → Phrygian).
Формула: Inverted[i] = 12 - original[11-i].
Chirality: Если scale = rotate(invert), achiral (compatible self); иначе chiral pairs для contrast в polyphony.

Evenness/Balance:
Evenness = variance(interval widths); low variance = stable для polyphony (e.g., whole-tone scale).
Формула: Для n-note scale, ideal spacing = 12/n; deviation sum((actual - ideal)^2).

Subset/Complement:
Complement = ~scale (bit-flip, mod 4096).
Combine: Polyphony через superset (e.g., chromatic как union всех) для max richness.

На основе анализа официальной документации KORG (включая Owner's Manual, Editor/Librarian Manual и Native Manual), Wavestate не имеет фиксированного "полного списка" предустановленных скейлов (scales) в виде именованных пресетов, как в Electribe (где есть 35 конкретных скейлов). Вместо этого Wavestate фокусируется на пользовательском редактировании скейлов и тюнингов, с поддержкой импорта файлов в формате Scala (.scl) для микротональных тюнингов. Это позволяет создавать тысячи вариаций, но предустановленный список отсутствует в мануалах — скейлы редактируются вручную или импортируются.
"Более 100 тюнингов" может относиться к:

128 Note Scale режиму, где можно индивидуально тюнинговать каждую из 128 MIDI-нот (что больше 100), не повторяясь на октавах.
Поддержке Scala файлов: архив Scala содержит более 5000 тюнингов, но в Wavestate они импортируются пользователем, а не предустановлены.
Общему количеству возможных комбинаций в редакторе (cents и semitones для каждой ноты).

Скейлы в Wavestate — это объекты (.wsscale), которые можно создавать, дублировать и редактировать в Editor/Librarian. Нет упоминаний о конкретных предустановленных именах скейлов (типа Ionian, Dorian и т.д. из Electribe), но можно воспроизвести любые стандартные скейлы вручную.
Типы Скейлов в Wavestate
Wavestate поддерживает три типа скейлов (types), каждый с возможностью тюнинга в cents (±100) и semitones (±127) для каждой ноты. Вот полный "список" типов из мануала:

Octave Scale
Стандартный 12-нотный скейл, повторяющийся каждую октаву.
Подходит для диатонических, модальных или этнических скейлов (например, Major, Minor, Pentatonic).
Тюнинг: 12 нот, root key (C...B).
Пример использования: Равномерная темперация (Equal Temperament) или Just Intonation.

Octave Scale, A=Master Tune
Аналогично предыдущему, но автоматически подстраивает скейл так, чтобы нота A соответствовала Master Tune (например, A=440 Hz), независимо от ключа.
Полезно для неравномерных темпераций, где A фиксирована.
Тюнинг: 12 нот, с автоматической корректировкой.

128 Note Scale
Полностью кастомный скейл с отдельным тюнингом для каждой из 128 MIDI-нот (C-2 до G8).
Не повторяется на октавах, подходит для микротональных, экспериментальных или неоктавных скейлов (например, Bohlen-P, Wendy Carlos scales).
Это дает "более 100 тюнингов" — точно 128 индивидуальных настроек нот.


Параметры Тюнинга для Всех Типов

Cents: ±100 (тонкая настройка в центах; 1 цент = 1/100 полутона).
Semitones: ±127 (сдвиг в полутонах, до полного MIDI-диапазона).
Key: C...B (root для повторяющихся скейлов).
Transpose: Transposes Scale или Within Scale (как транспозиция взаимодействует со скейлом).
Master Tune: ±50 центов (общая настройка, A4 = 427.47–452.89 Hz).
Global Scale: On/Off (применяет скейл глобально или per-performance).

Хорошо нам нужно реализовать программу идеальную для джема, какие есть open source решения для алгоритмического определения scale ?
 

Mode Wheel (например, RubberDave's или аналогичные cardboard tools).

  * Структура: Два диска: внешний — modes (Ionian-Dorian-...-Locrian); внутренний — 12 нот. Rotate для relative modes.
  * Сочетание шкал: Находит equivalents (e.g., G Mixolydian = C Major notes). Для полифонии: Switch modes в разных voices (e.g., bass в Aeolian, melody в Dorian) для richness.
  * Пример: В прог-роке — polyphonic layers из diatonic modes для complex textures.

Musicians’ Theory Wheel (коммерческий инструмент).

  * Структура: Rotate для matching chords/scales; включает circle of fifths, progressions.
  * Сочетание шкал: Instant harmonic compatibility (e.g., scales с shared chords). Для полифонии: Combine via progressions (ii-V-I в different modes).Open Source Решения для Алгоритмического Определения Музыкальных Шкал
Для реализации идеальной программы для музыкальных джем-сессий (jam sessions), где нужно определять шкалы (scales) в реальном времени на основе сыгранных нот или аудио, подойдут библиотеки, которые анализируют pitch (высоту тона), chroma features (хроматические характеристики) или tonal descriptors. Это позволяет сопоставлять последовательность нот с известными шкалами (например, major, minor, pentatonic). "Определение шкалы" часто сводится к key detection (определение тональности), так как key включает scale (e.g., C Major scale).
Полный список основан на анализе доступных open source инструментов. Они бесплатны, с открытым кодом (обычно GPL или MIT), и подходят для Python/C++ разработки. Большинство поддерживают real-time анализ через streaming audio (микрофон или MIDI). Я разделил на библиотеки для аудио анализа (из звука) и symbolic (из нот/MIDI). Для джема рекомендую комбинировать с MIDI input (e.g., via Mido) или аудио capture (e.g., PyAudio).
1. Essentia (C++ с Python bindings, Affero GPLv3)

Описание: Библиотека для аудио анализа и music information retrieval (MIR). Прямо поддерживает key/scale detection: алгоритм KeyExtractor определяет key (e.g., C Major) и scale (major/minor) на основе HPCP (harmonic pitch class profile, chroma аналог). Использует pitch salience, predominant melody extraction для полифонии. Оптимизирована для real-time (low-latency, streaming mode).
Применение в джеме: Анализирует live audio, определяет prevailing scale для импровизации. Можно интегрировать с Gaia для ML-классификации scales.
Пример использования (Python):Pythonfrom essentia.standard import *
audio = MonoLoader(filename='input.wav')()
key_extractor = Key()
hpcp = HPCP()(Spectrum()(Windowing()(audio)))
key, scale, strength = key_extractor(hpcp)
print(f"Detected: {key} {scale}")Для real-time: Используйте streaming mode с AudioLoader.
Репозиторий: GitHub - MTG/essentia
Плюсы: Robust, expandable, Vamp plugin для визуализации. Минусы: C++ core, но Python удобен.

2. audioFlux (C с Python bindings, MIT)

Описание: Библиотека для deep learning аудио анализа. Включает pitch detection (YIN, STFT, HPS etc.) и MIR features (onset, HPSS). Chroma из CQT/BFT помогает в scale detection: анализируйте pitch classes для matching с scales.
Применение в джеме: Real-time pitch tracking + chroma для live scale recognition (e.g., windowed analysis).
Пример использования (Python):Pythonimport audioflux as af
import numpy as np
spec_arr, _ = af.BFT(num=84, samplate=32000, radix2_exp=12, slide_length=1024).bft(audio_arr)
chroma_arr = af.Chroma(num=12, samplate=32000).chroma(spec_arr)
# Match chroma to scales (custom logic)
Репозиторий: GitHub - libAudioFlux/audioFlux
Плюсы: DL-based, real-time transforms. Минусы: Требует custom code для full scale matching.

3. Aubio (C с Python bindings, GPL)

Описание: Инструменты для аудио labelling. Pitch tracking (YIN fast, spectral convolution), onset/beat detection, MFCC. Не прямой scale detector, но pitch + chroma-like features позволяют строить scale analysis.
Применение в джеме: Real-time pitch extraction для monophonic instruments; комбинируйте с custom matching к scales.
Пример использования (Python):Pythonimport aubio
pitch_o = aubio.pitch("yinfast", 2048, 1024, samplerate)
pitch = pitch_o(samples)
# Analyze pitches over time for scale
Репозиторий: GitHub - aubio/aubio
Плюсы: Low-delay, для live audio/MIDI. Минусы: Базовый; нужна доп. логика.

4. Librosa (Python, MIT)

Описание: Для аудио/музыки. Chroma features (chroma_stft, chroma_cqt), pitch shift, key estimation via chroma. Не direct scale, но chroma vector matching к scale templates (e.g., major/minor profiles).
Применение в джеме: Real-time с PyAudio; анализируйте frames для scale.
Пример использования:Pythonimport librosa
y, sr = librosa.load('audio.wav')
chroma = librosa.feature.chroma_stft(y=y, sr=sr)
# Correlate with scale templates
Репозиторий: GitHub - librosa/librosa
Плюсы: Легкая в использовании. Минусы: Не оптимизирована для ultra-low latency.

5. RecogNotes (Python, MIT)

Описание: Real-time note detector с CREPE (CNN pitch tracker). Генерирует sheets, но не напрямую scales; pitches можно аггрегировать для scale fitting.
Применение в джеме: Live mic input → notes → custom scale detection.
Репозиторий: GitHub - orgurar/recognotes
Плюсы: Web/CLI интерфейс. Минусы: Фокус на notes, не scales.

Другие

pyAudioAnalysis (Python, Apache): Feature extraction (chroma, MFCC), classification. Для scale: chroma-based ML.
Nootka (C++, GPL): App для обучения с pitch detection; можно адаптировать для jam.
Musicaiz (Python, MIT): Symbolic analysis; для MIDI scales, не real-time audio.
musical-scales (Python, MIT): Retrieval scales по mode/note; не detection, но helper.

Рекомендации для Реализации Программы

Для джема: Начните с Essentia или Aubio + Python (PyAudio для mic). Добавьте ML (e.g., scikit-learn) для matching pitches к 100+ scales (из вашего предыдущего списка). Используйте MIDI для точности (Mido lib).
Шаги: 1) Capture audio/MIDI; 2) Extract pitches/chroma; 3) Match к scale templates (e.g., interval vectors); 4) Display/UI (e.g., Tkinter или web с Flask).
Если нужно ML: Обучите на datasets (e.g., GiantMIDI) для предсказания scales.
Протестируйте на GitHub примерах; все free и expandable. Если нужны детали, уточните!
